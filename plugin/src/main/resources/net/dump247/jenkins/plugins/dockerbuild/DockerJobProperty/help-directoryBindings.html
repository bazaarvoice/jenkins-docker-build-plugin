Map directory on host machine (machine running the jenkins agent) to a path in the docker container.

Format: hostPath[:containerPath[:access]]

hostPath is the path on the jenkins agent to map into the container.

containerPath is the path to map to in the container. If not specified, the hostPath is used. The directory does not need to exist in the container image. It will be created if needed. If the path already exists, the new file system will hide the existing directory.

access determines the access the container has to the mapped directory. Can be r for read-only and rw for read-write. Default is read-write.

Blank lines or lines that start with # are ignored.

Environment variables from the jenkins agent can be referenced in the host or container paths with ${VARIABLE} or ${env.VARIABLE}. System properties from the jenkins agent can be referenced in the host or container paths with ${sys.variable}.

Examples:

# Same as /host/path:/host/path:rw /host/path # Same as /host/path:/host/path:r /host/path::r # Same as /host/path:/container/path:rw /host/path:/container/path # Makes /container/path read-only for the container /host/path:/container/path:r
