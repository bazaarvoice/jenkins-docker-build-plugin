Bind directories from the host machine to paths in the container. If the parent directories do not exist in the container, they will be created. If a directory already exists with the same path in the container, the host directory will be mounted at the given path and will hide the original directory on the container filesystem.

A directory can be mapped read-only. The files will still display the same permissions as the host system, but any write attempts will fail.

Format

Blank lines and lines that start with a # are ignored. Default access is read-only.

FormatDescription
/a/pathMap /a/path/ in the host to /a/path/ in the container, read-only.
/a/path:rMap /a/path/ in the host to /a/path/ in the container, read-only.
/a/path:rwMap /a/path/ in the host to /a/path/ in the container with write access.
/host/path:/container/pathMap /host/path/ in the host to /container/path/ in the container, read-only.
/host/path:/container/path:rMap /host/path/ in the host to /container/path/ in the container, read-only.
/host/path:/container/path:rwMap /host/path/ in the host to /container/path/ in the container with write access.
