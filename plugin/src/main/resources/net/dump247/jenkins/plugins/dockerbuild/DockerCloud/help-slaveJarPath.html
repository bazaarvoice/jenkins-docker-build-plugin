Path to the slave jar in the job container. If not specified, the slave jar is downloaded each time a job starts. It can help improve job startup time to store the slave jar on the host and map it into each container.

To properly set it up:

  1. Create a cron job on the host that regularly downloads the slave jar
    • For example, to /var/lib/jenkins/slave.jar, but you can place it anywhere
    • The slave jar URL is http://JENKINS_HOST/jnlpJars/slave.jar
    • No authentication should be required to get the jar
    • Make sure to retrieve the slave jar immediately to get an initial local copy
    • An example script can be found at github.com/dump247/jenkins-docker-build-plugin/blob/master/deploy/sync_agent
  2. Add the directory that contains the slave jar in the directory mappings section
    • /var/lib/jenkins
  3. Set this option to the path to the slave jar in the container
    • /var/lib/jenkins/slave.jar
